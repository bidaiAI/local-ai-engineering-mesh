# Workflows and Combos

## What this system can actually do

Compared with a fragmented tool-by-tool workflow, this system can:
- keep one operating law across multiple tools
- preserve project memory across longer tasks
- route work to the best-fit endpoint instead of forcing one tool to do everything
- require evidence before completion
- turn repeated patterns into reusable structure
- support single-tool, dual-tool, and full-mesh adoption

## Original workflow vs engineered mesh

### Original fragmented workflow
- switch tools ad hoc
- repeat instructions manually
- lose context between tools
- quality bar drifts
- release and publish are loosely controlled

### Engineered mesh workflow
- one shared law at the top
- specialized endpoints underneath
- project runtime memory and state
- explicit evidence and release discipline
- reusable templates and role patterns

## End-to-end workflow

1. start from shared law
2. bootstrap the project runtime
3. choose the active endpoint for the current phase
4. execute with local rules and project memory
5. verify with evidence
6. publish or deploy through gates
7. promote repeated wins into reusable templates or skills

## Highest-performance combinations

### 1. Codex only
Best for:
- single-tool users
- local execution-first workflows
- governed implementation and verification

### 2. Codex + Cursor
Best for:
- coding-heavy work
- local execution plus IDE speed
- practical day-to-day software delivery

### 3. Codex + Claude
Best for:
- stronger workflow methodology
- design and review plus governed execution
- long-horizon engineering tasks

### 4. Codex + Cursor + Claude
Best for:
- highest-performance engineering combination for most software work
- Claude for methodology and review
- Cursor for editor-native delivery
- Codex for execution and governance

### 5. Full mesh: Codex + Claude + Cursor + Antigravity
Best for:
- the broadest capability surface
- software delivery plus research plus browser-heavy work
- users who want one operating law across multiple specialized endpoints
