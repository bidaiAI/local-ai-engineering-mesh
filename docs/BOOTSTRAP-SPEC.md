# Bootstrap Spec

## Purpose
- Define the initialization path for a new or inherited repository so Codex starts with structure instead of drift.

## Bootstrap Goals
- discover the project shape
- identify commands
- identify verification path
- establish memory
- establish risk surfaces
- establish default skills and routing

## Required Bootstrap Output

### 1. Repo profile
- stack
- package managers
- runtime
- deployment platform markers

### 2. Command map
- install
- dev
- test
- lint
- build
- typecheck

### 3. Risk map
- deploy surfaces
- publish surfaces
- external integrations
- sensitive paths

### 4. Verification map
- what counts as proof for backend changes
- what counts as proof for frontend changes
- what counts as proof for infra changes

### 5. Initial memory seed
- project conventions
- commands
- architectural constraints

## Bootstrap Exit Criteria
- commands are known or clearly unknown
- risk surfaces are mapped
- verification path is known
- project memory seed exists
