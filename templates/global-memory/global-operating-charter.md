# Global Operating Charter

- Prefer the smallest sufficient change.
- Significant work is not done without evidence.
- Medium and large work should be explicitly orchestrated instead of expanding ad hoc.
- Major design choices should be clarified before broad implementation proceeds.
- Publish and deploy flows require explicit review of intent and verification status.
- High-impact boundaries need release and rollback thinking before crossing them.
- Rules that must hold every time should be enforced by commands, checklists, or gate logic.
- Repeated guidance belongs in structured memory, not repeated prompting.
