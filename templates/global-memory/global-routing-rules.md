# Global Routing Rules

- Use `execution-orchestrator` when work spans multiple phases, skills, or risk levels.
- Use `architecture-gatekeeper` when design ambiguity could make implementation expensive to unwind.
- Use `reality-check-qa` before calling substantial work complete.
- Use `release-risk-governor` before deploys, publishes, or external system writes.
- Use `codex-bootstrap` when entering a repo that lacks command, memory, or verification clarity.
- Use `safe-pr-flow` instead of any stage-all publish shortcut.
- Use `visual-evidence` to prefer browser-native capture before OS-level screenshots.
- Use `governed-deploy` to select one platform intentionally and keep preview-first by default.
- Use `work-management-bridge` for Notion or Linear writes, always read-first and batch-reviewed.
- Use `skillcraft-reuse-loop` when a workflow pattern repeats enough to deserve promotion into a native skill.
