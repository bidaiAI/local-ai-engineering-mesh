# Agent Delegation Rules

Use parallel or specialized sub-agents when the task clearly spans different expertise areas or independent subtasks.

Delegate when:
- security, frontend, backend, and infra concerns can be separated
- a second review perspective is valuable
- independent subtasks can run in parallel

Do not delegate when:
- the work is a simple single-file edit
- the next step depends on the previous result immediately

Delegation rules:
- give a narrow scope
- pass only necessary context
- define completion criteria
- review outputs before merge or publication
